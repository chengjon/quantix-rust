quantix import from-text

命令路径：quantix import from-text

从文本解析股票代码/名称

用法

quantix import from-text --text <TEXT>

参数

名称值说明
-t, --textTEXT文本内容 (股票代码或名称，逗号/空格/换行分隔)
-h, --help-Print help

使用示例

quantix import from-text --text "000001, 招商银行, GZMT"

功能说明

命令会按换行、制表符、逗号、中英文分号以及空格对文本进行切分，并尝试把每个 token 解析成股票代码、名称或拼音首字母。像 000001 平安银行 这样的“代码 名称”对会优先提取代码一侧；解析成功的项目会按代码去重，失败 token 会进入错误列表。

quantix import resolve

命令路径：quantix import resolve

解析股票名称/代码

用法

quantix import resolve --input <INPUT>

参数

名称值说明
-i, --inputINPUT输入文本 (代码或名称)
-h, --help-Print help

使用示例

quantix import resolve --input PAYH

功能说明

这是单条解析入口，适合快速判断一个输入能否被当前内置映射识别。解析器会依次尝试精确代码、精确名称、拼音首字母和模糊名称匹配，并返回匹配方式与置信度；如果映射表中不存在对应股票，它不会走远程检索，只会打印本地提示帮助用户调整输入格式。

quantix import market-manifest

命令路径：quantix import market-manifest

解析 miniQMT 发布的市场数据 manifest（仅 dry-run，不导入）

用法

quantix import market-manifest [OPTIONS] --manifest <MANIFEST> --dataset-version <DATASET_VERSION> --artifact-type <ARTIFACT_TYPE>

参数

名称值说明
--manifestMANIFESTmanifest JSON 文件路径
--dataset-versionVERSION期望的数据集版本
--artifact-typeTYPE期望的发布物类型（如 parquet/json）
--schema-versionVERSION可选 schema version 约束
--artifact-hashHASH可选发布物 hash 约束
--verify-artifact-file-读取本地 artifact 文件并重算 hash（仅支持本地路径 / file://）
--comparison-reference-artifactPATH本地 reference artifact，用于 double-read comparison
--comparison-source-of-truth-summaryPATHsource-of-truth 汇总 JSON
--comparison-clickhouse-urlURLClickHouse HTTP URL
--comparison-clickhouse-databaseDBClickHouse database [default: quantix]
--comparison-clickhouse-userUSERClickHouse user [default: default]
--comparison-clickhouse-passwordPASSClickHouse password [default: ]
--comparison-clickhouse-tableTABLEClickHouse 表名
--comparison-clickhouse-dataset-version-columnCOLdataset_version 列名 [default: dataset_version]
--comparison-clickhouse-symbol-columnCOLsymbol 列名 [default: symbol]
--comparison-clickhouse-date-columnCOLdate 列名 [default: date]
--regression-report-outputPATH输出 Quantix raw regression report JSON
--evidence-outputPATH输出 miniQMT controlled evidence JSON
--consumer-build-commitCOMMITQuantix consumer build commit [env: QUANTIX_CONSUMER_BUILD_COMMIT]
--database-targetTARGET数据库目标；当前默认 dry-run-only [default: dry-run-only]
-h, --help-Print help

使用示例

quantix import market-manifest --manifest manifest.json --dataset-version v2 --artifact-type parquet
quantix import market-manifest --manifest manifest.json --dataset-version v2 --artifact-type parquet --verify-artifact-file --regression-report-output report.json
quantix import market-manifest --manifest manifest.json --dataset-version v2 --artifact-type parquet --comparison-reference-artifact /data/ref.parquet --evidence-output evidence.json

功能说明

用于接收 miniQMT 发布的市场数据 manifest 并做解析校验。当前默认 --database-target dry-run-only，仅做 dry-run 校验不执行实际数据库写入。核心校验包括：验证 manifest JSON 结构、校验 dataset-version 与 artifact-type 是否匹配期望值、可选 hash 校验（--artifact-hash 或 --verify-artifact-file）。支持三种 opt-in double-read comparison 模式：本地 reference artifact 对比、source-of-truth 汇总 JSON 对比、以及 ClickHouse 直连只读对比。--regression-report-output 和 --evidence-output 分别输出 Quantix 侧回归报告和 miniQMT 侧受控证据 JSON。
