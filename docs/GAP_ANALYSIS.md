# Quantix Rust 功能缺口分析

> 本文不是功能状态注册表；功能状态、已设计/待实现项和可用边界以根目录 `FUNCTION_TREE.md` 的状态注册表行为准。

基于量化交易系统的完整生命周期（数据 → 研究 → 策略 → 执行 → 风控 → 运维），分析当前功能缺口。

---

## 一、数据层缺口

### 1.1 基本面数据 ❌ 缺失

```
当前状态: 仅支持行情数据 (OHLCV)
缺失功能:
├── 财务数据
│   ├── 资产负债表
│   ├── 利润表
│   ├── 现金流量表
│   └── 财务指标 (ROE/ROA/PE/PB等)
├── 公司公告
│   ├── 业绩预告/快报
│   ├── 重大事项公告
│   └── 分红送转
└── 分析师数据
    ├── 评级数据 (买入/持有/卖出)
    └── 目标价预测
```

**影响**: 无法实现基本面量化策略、价值投资策略

### 1.2 另类数据 ❌ 缺失

```
缺失功能:
├── 新闻舆情
│   ├── 新闻情感分析
│   ├── 热点事件追踪
│   └── 关联个股分析
├── 社交媒体
│   ├── 雪球/东方财富讨论热度
│   └── 散户情绪指标
├── 龙虎榜数据
│   ├── 游资/机构席位追踪
│   └── 异常交易监控
└── 融资融券数据
    ├── 融资余额变化
    └── 融券余额变化
```

**影响**: 无法捕捉另类因子、情绪驱动策略

### 1.3 Level-2 数据 ❌ 缺失

```
当前状态: 仅支持 Level-1 快照数据
缺失功能:
├── 逐笔成交 (Tick Data)
├── 买卖盘口 (10档/50档)
├── 委托队列
└── 成交分布
```

**影响**: 无法实现高频策略、微观结构策略

### 1.4 衍生品数据 ❌ 缺失

```
缺失功能:
├── 期权数据
│   ├── 期权链 (Option Chain)
│   ├── 隐含波动率
│   └── Greeks (Delta/Gamma/Vega/Theta)
├── 期货数据
│   ├── 股指期货 (IF/IC/IM)
│   ├── 基差数据
│   └── 仓位数据
└── ETF 数据
    ├── ETF 申赎清单
    └── ETF 溢价率
```

**影响**: 无法实现套利策略、对冲策略

## 二、策略层缺口

### 2.1 策略参数优化 ❌ 缺失

```
当前状态: 手动调整策略参数
缺失功能:
├── 参数网格搜索 (Grid Search)
├── 遗传算法优化
├── 贝叶斯优化
├── Walk-Forward 分析
└── 参数稳定性分析
```

**影响**: 策略参数选择依赖经验，缺乏系统性优化

### 2.2 多策略组合 ⚠️ 部分缺失

```
当前状态: 支持多策略运行，但缺乏组合管理
缺失功能:
├── 策略权重分配
├── 策略相关性分析
├── 组合层面资金分配
├── 动态权重调整
└── 策略切换机制
```

**影响**: 多策略协同效果差，资金利用效率低

### 2.3 因子库系统 ❌ 缺失

```
缺失功能:
├── 因子定义框架
├── 因子计算引擎

│   └── 因子衰减分析
├── 因子正交化
└── 因子组合 (多因子模型)
```

**影响**: 无法系统化管理和复用量化因子

### 2.4 机器学习框架 ❌ 缺失

```
缺失功能:
├── 特征工程
│   ├── 特征提取
│   ├── 特征选择
│   └── 特征标准化
├── 模型训练
│   ├── 监督学习 (分类/回归)
│   ├── 无监督学习 (聚类/降维)
│   └── 强化学习
├── 模型评估
│   ├── 交叉验证
│   ├── 样本外测试

└── 模型服务化

    └── 在线推理服务
```

## 三、执行层缺口

### 3.1 算法交易 ❌ 缺失

```
当前状态: 简单限价/市价单
缺失功能:
├── TWAP (时间加权平均价格)
├── VWAP (成交量加权平均价格)
├── POV (参与率)
├── IS (实施短缺)
├── 冰山订单
└── 智能路由
```

**影响**: 大单执行滑点大，冲击成本高

### 3.2 滑点与冲击成本 ⚠️ 部分缺失

```
当前状态: 模拟交易有基础滑点模型
缺失功能:
├── 滑点建模
│   ├── 历史滑点分析
│   ├── 滑点预测模型
│   └── 滑点归因
├── 冲击成本分析
│   ├── 临时冲击
│   └── 永久冲击
└── 最优执行路径
```

**影响**: 无法评估真实执行成本

### 3.3 多券商/多账户 ❌ 缺失

```
当前状态: 单一账户模式
缺失功能:
├── 多券商接入
│   ├── 券商适配器抽象
│   ├── 统一订单接口
│   └── 资金聚合视图
├── 多账户管理
│   ├── 账户组管理
│   ├── 资金调拨
│   └── 账户间套利
└── 订单路由
    ├── 智能拆单
    └── 最优执行场所选择
```

**影响**: 无法管理多账户、多券商场景

### 3.4 真实实盘接口 ⚠️ 部分实现

```
当前状态: 已有受能力门控的 `qmt_live` 真实提交通道，但通用 `live` 语义和更完整的多券商实盘能力仍未完成
缺失功能:
├── 通用 `target_mode=live` 统一语义
├── 恒生 UFX 接口
├── 证券公司 API 接入
└── 更完整的撤单/改单与回报闭环
```

**影响**: 已能通过 guarded `qmt_live` 进入真实提交，但仍无法宣称“通用自动化实盘交易能力已完整交付”

## 九、优先级建议

### P0 - 核心功能 (必须完成)

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P0 | 真实实盘接口 | 没有它就不是真正的交易系统 |
| P0 | 算法交易 (TWAP/VWAP) | 降低大单冲击成本 |
| P0 | 多账户管理 | 实际投资场景必需 |
| P0 | 实时风险监控 | 防止重大损失 |
| P0 | Web 监控面板 | 运维效率 |

### P1 - 重要功能 (建议完成)

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P1 | 因子库系统 | 提升策略研发效率 |
| P1 | 策略参数优化 | 提升策略质量 |
| P1 | 基本面数据 | 支持更多策略类型 |
| P1 | 压力测试 | 风险管理 |
| P1 | 绩效归因 | 理解收益来源 |
| P1 | VaR 计算 | 量化风险评估 |

### P2 - 增强功能 (可选完成)

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P2 | 机器学习框架 | 支持 AI 策略 |
| P2 | Level-2 数据 | 支持高频策略 |
| P2 | 统计套利 | 市场中性策略 |
| P2 | 自动化报告 | 提升效率 |
| P2 | 灾备机制 | 系统可靠性 |
| P2 | 安全机制 | 合规要求 |

### P3 - 高级功能 (长期规划)

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P3 | 期权/期货数据 | 支持衍生品策略 |
| P3 | 另类数据 | 增强信号 |
| P3 | 资产配置模型 | 机构级功能 |
| P3 | 合规风控 | 监管要求 |

## 十一、下一步建议

基于当前项目状态和实际使用场景，建议按以下顺序推进：

### 短期 (1-2周)
1. **真实实盘接口** - QMT 真实下单能力
2. **Web 监控面板** - 提升运维体验
3. **策略参数优化** - 网格搜索 + Walk-Forward

### 中期 (1-2月)
1. **因子库系统** - 因子定义、计算、存储、检验
2. **算法交易** - TWAP/VWAP 实现
3. **多账户管理** - 账户组、资金聚合
4. **VaR 计算** - 历史模拟法

### 长期 (3-6月)
1. **机器学习框架** - 特征工程 + 模型训练
2. **基本面数据** - 财务数据接入
3. **压力测试** - 历史情景 + 假设情景
4. **灾备与安全** - 高可用架构
