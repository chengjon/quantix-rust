Date: 2026-06-21

## Summary

P0.4c enriches the qmt_live-local error taxonomy seeded in P0.3d. The change remains limited to `QmtLiveErrorCategory` in `src/execution/qmt_task_submit_service.rs` and its contract tests.

The implementation uses only information already present in current task-contract surfaces:

- typed `BridgeError` variants
- qmt task identity mismatch messages generated by `qmt_task_submit_service`
- existing `QmtTaskResolvedResult.latest_status`

No global error response, request diagnostics, CLI output, bridge protocol, response schema, storage schema, order status, execution adapter trait, or submit/query/cancel behavior changed.

## Implemented Contract

`QmtLiveErrorCategory::from_bridge_error` now distinguishes typed bridge failure surfaces instead of collapsing every bridge-side error into `bridge_failure`:

- `BridgeError::Config` -> `local_validation_rejected`
- `BridgeError::Timeout` -> `bridge_timeout`
- `BridgeError::Unavailable` -> `bridge_unavailable`
- `BridgeError::Unauthorized` -> `bridge_auth_failed`
- `BridgeError::UnsupportedContractVersion` -> `bridge_unsupported_contract_version`
- `BridgeError::UnsupportedMethod` -> `bridge_unsupported_method`
- `BridgeError::Protocol` -> `bridge_protocol_violation`
- `BridgeError::Http` -> `bridge_http_failure`
- identity mismatch `BridgeError::InvalidResult` -> `task_identity_mismatch`
- other `BridgeError::InvalidResult` -> `bridge_invalid_result`

`QmtLiveErrorCategory::from_task_result` behavior remains unchanged:

- `OrderStatus::Rejected` -> `broker_rejected`
- `OrderStatus::Unknown` -> `broker_unknown_state`
- other statuses -> `None`

The existing umbrella and reserved categories remain available for future wiring:

- `bridge_failure`
- `local_risk_gate_rejected`
- `manual_intervention_required`
