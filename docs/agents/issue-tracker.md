# Issue Tracker

Matt Pocock skills should use GitHub Issues for project work items in this repository.

## Default Repository

Use `chengjon/quantix-rust` for new code-audit, remediation, and architecture-review issues unless the maintainer explicitly routes a task elsewhere.

Repository remotes observed during setup:

| Remote | URL | Use |
|---|---|---|
| `origin` | `git@github.com:chengjon/quantix-rust.git` | Default issue tracker for this repository |
| `quantix` | `git@github.com:chengjon/quantix.git` | Historical or alternate tracker; do not use for new issues unless requested |
| `mystocks` | `git@github.com:chengjon/mystocks.git` | Historical or alternate tracker; do not use for new issues unless requested |

## Creation Rules

- Do not create external issues until the maintainer confirms the issue batch.
- Use `to-issues` to draft tracer-bullet issues before publishing.
- Use `triage` to assign one category role and one state role to each issue.
- Every triage comment posted by an agent must start with:

```text
This was generated by AI during triage.
```

## Required Context

Before publishing issues from an audit report, read:

- `docs/reports/CODE_AUDIT_FINAL_2026-05-15.md`
- `docs/reports/IMPECCABLE_AUDIT_CODE_AUDIT_EXECUTION_SPEC_2026-05-15.md`
- `docs/CODE_AUDIT_EVIDENCE/findings.csv`
- `docs/CODE_AUDIT_EVIDENCE/cargo-gates.md`

Do not treat issue state as feature status. Feature status remains governed by `FUNCTION_TREE.md`.
