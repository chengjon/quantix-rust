二、核心缺口分析与优化建议
缺口1：插件管理命令完全缺失
当前状态：有 strategy list/run，但没有插件生命周期管理。

建议新增：

bash
# 插件管理（新增）
quantix plugin list                        # 列出已安装插件
quantix plugin install --path ./ma_strategy.wasm --name ma_cross
quantix plugin remove --id ma_cross
quantix plugin enable --id ma_cross
quantix plugin disable --id ma_cross
quantix plugin test --id ma_cross --code 600519  # 单元测试模式

# 策略与插件关联（重构现有）
quantix strategy list --plugin-id ma_cross       # 列出某插件的策略实例
quantix strategy create --plugin ma_cross --name "我的MA策略" --params '{"period":20}'
理由：插件是架构核心，必须暴露给用户管理。

缺口2：数据源配置命令缺失
当前状态：data query 直接查询，但不知道数据来自哪里。

建议新增：

bash
# 数据源管理
quantix data source list                           # 列出已配置数据源
quantix data source add --name tdx --type local --path /mnt/tdx
quantix data source add --name akshare --type http --endpoint https://akshare.com
quantix data source set-default --name tdx
quantix data source test --name akshare            # 测试连接

# 数据采集（对应 data collector）
quantix data collect --source akshare --code 600519 --start 20240101 --end 20241231
quantix data subscribe --ws wss://push.tdx.com --code 600519,000001
理由：数据底座是独立的，CLI必须支持多数据源切换。

缺口3：回测能力不足（缺关键参数）
当前状态：analyze backtest --id 只能查看已有回测，无法发起新回测。

建议增强：

bash
# 发起回测（新增）
quantix backtest run --strategy ma_cross --code 600519 \
    --start 20240101 --end 20241231 \
    --capital 1000000 \
    --commission 0.0003 --stamp-duty 0.001 \
    --slippage 0.0001 \
    --parallel 4                     # 并行回测多股票

# 参数优化（新增）
quantix backtest optimize --strategy ma_cross --code 600519 \
    --param "period:5,10,20,30" \
    --param "fast:5,10" --param "slow:20,30" \
    --metric sharpe_ratio --top 10

# 回测对比（新增）
quantix backtest compare \
    --run-id backtest_20241215_001 \
    --run-id backtest_20241215_002 \
    --metrics "total_return,max_drawdown,sharpe_ratio"
理由：回测引擎是核心能力，必须提供完整的CLI入口。

缺口4：实盘交易与模拟交易未分离
当前状态：trade buy/sell 是模拟交易，但没有实盘命令。

建议分离：

bash
# 模拟交易（保持现有）
quantix paper buy 600519 --price 1680 --volume 100
quantix paper sell 600519 --price 1700 --volume 50
quantix paper account                         # 模拟账户状态

# 实盘交易（新增）
quantix live status                           # 实盘连接状态
quantix live buy 600519 --price 1680 --volume 100 --confirm
quantix live sell 600519 --price 1700 --volume 50 --confirm
quantix live order list                       # 今日委托
quantix live order cancel --order-id 12345
quantix live position                         # 实盘持仓
quantix live account                          # 实盘资金

# 桥接配置（已有 execution bridge，重命名）
quantix live bridge status                    # 原 execution bridge status
quantix live bridge qmt account               # 原 execution bridge qmt-account
理由：模拟和实盘必须明确区分，避免误操作。

缺口5：绩效分析与策略对比不足
当前状态：risk pnl 只能看当日盈亏，无历史对比。

建议增强：

bash
# 绩效分析（新增）
quantix performance report --strategy ma_cross --start 20240101 --end 20241231
quantix performance compare --strategies ma_cross,rsi_divergence,breakout \
    --metrics "total_return,sharpe_ratio,max_drawdown,win_rate" \
    --output table

# 归因分析（新增）
quantix performance attribution --run-id backtest_001 \
    --factors "market,size,value,momentum" \
    --output json

# 历史排名（新增）
quantix performance leaderboard --metric sharpe_ratio --limit 10
理由：策略横向比较是量化系统的核心价值。

缺口6：参数管理与配置版本化缺失
当前状态：策略参数散落在各个命令中，无统一管理。

建议新增：

bash
# 参数模板管理
quantix param template list                    # 列出参数模板
quantix param template create --name ma_standard --params '{"period":20,"fast":5,"slow":20}'
quantix param template apply --template ma_standard --strategy ma_cross

# 配置版本管理
quantix config version list                    # 列出配置版本
quantix config version save --name "2024年12月生产配置"
quantix config version load --name "2024年12月生产配置"
quantix config diff --version v1 --version v2
理由：量化策略需要可复现、可回滚的配置管理。
