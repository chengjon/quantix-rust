# Phase 1 Plan 02: Recovery And Reconciliation Closure

## Objective
- **What:** Close the gap between existing reconciliation scaffolding and the roadmap claim that open-order/account reconciliation is part of Phase 1.
- **Why:** Hardening is incomplete if the system can model non-final orders but cannot reliably surface and reconcile them.
- **Output:** Hardened scanner/reconciliation behavior with system-level tests tying runtime order state to recovery outcomes and account deltas.
