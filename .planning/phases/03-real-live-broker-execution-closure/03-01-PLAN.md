# Phase 3 Plan 01: Live Adapter Contract And Broker Boundary Lock

## Objective
- **What:** Lock the real-live adapter contract and the broker path boundary around `qmt_live` versus generic `live`.
- **Why:** `LIV-01` is not satisfied by scattered code alone; the repo needs a test-backed contract that says which path is allowed to submit real orders.
- **Output:** Hardened boundary semantics plus regressions that fail if generic `live` starts behaving like a real-submit path.
