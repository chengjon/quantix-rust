# Phase 3 Plan 02: Explicit QMT Live Gate Hardening

## Objective
- **What:** Harden the explicit QMT real-submit gate and its failure semantics across adapter, daemon, and manual bridge paths.
- **Why:** `LIV-02` requires more than having a gate helper; it requires that the real-submit path stays explicitly guarded and regression-backed.
- **Output:** A minimum live-submit gate contract plus integration tests that prove misconfigured bridge states fail safely and visibly.
