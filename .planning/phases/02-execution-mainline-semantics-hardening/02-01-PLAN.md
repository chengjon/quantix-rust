# Phase 2 Plan 01: Request Vs Order Semantics Lock

## Objective
- **What:** Tighten operator-facing request and daemon formatting so request lifecycle state cannot be mistaken for order terminal state.
- **Why:** `SEM-01` fails if `completed` still reads like "filled" or "done" in compact CLI output.
- **Output:** Updated request/daemon formatter behavior with regression tests that lock the intended semantics.
