# Phase 2 Plan 02: Request Diagnostics And Observability Hardening

## Objective
- **What:** Tighten daemon and request payload diagnostics so operators can localize failed or stuck requests from CLI output and stored request payloads.
- **Why:** `SEM-02` requires more than correct status transitions; it requires actionable observability.
- **Output:** Normalized request diagnostics plus regression coverage for failure, cancel, and in-progress observability.
